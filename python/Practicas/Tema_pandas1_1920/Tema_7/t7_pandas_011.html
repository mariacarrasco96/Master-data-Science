Las Series y DataFrames de Pandas¶

La librera Pandas proporciona estructuras de datos de alto nivel que permiten representar series y tablas de datos, as como herramientas diseadas especficamente para un tratamiento de datos rpido y sencillo. Est construida sobre los fundamentos de NumPy, adaptando las operaciones sobre arrays a las nuevas estructuras definidas en Pandas. Resulta especialmente til cuando hay que trabajar con datos heterogneos representados de forma tabular.

Empez a desarrollarse en el ao 2008 por Wes McKinney (McKinney, 2012) en un intento por conseguir una herramienta flexible y de alto rendimiento para el tratamiento de datos financieros. En el ao 2012, Chang She comenz a participar en el desarrollo de la librera. Actualmente Pandas forma parte de un proyecto en el que participan muchos desarrolladores.

Pandas es actualmente una de las libreras de alto nivel de Python ms utilizadas en las primeras etapas del proceso de anlisis de datos, especialmente apropiada en aplicaciones estadsticas.

Toda la informacin oficial de la librera se encuentra accesible en el enlace http://pandas.pydata.org.

Estructuras de Datos en Pandas¶

En Pandas disponemos de varias estructuras de datos y mtodos que nos ayudarn a manipular dichas estructuras de forma sencilla y eficiente. Las dos estructuras de datos principales vienen representadas por las clases Series y DataFrame, capaces de representar series de datos y datos en forma tabular respectivamente.

Series¶

Un objeto del tipo Series puede verse como un objeto de tipo array de una dimensin que contiene datos de un cierto tipo y tiene asociado un ndice. Este ndice es un array de etiquetas cuyo tamao coincide con el tamao del array de datos.

A continuacin creamos un objeto de tipo Serie mediante la funcin pd.Series y una lista de elementos:

In [3]:
s = pd.Series([10, 20, 30, 40, -10])   
s
Out[3]:
0    10
1    20
2    30
3    40
4   -10
dtype: int64

Como podemos observar, la serie s contiene elementos de tipo int64 (columna de la derecha). Los elementos de la serie estn indexados mediante un array de enteros comenzando desde el 0 (columna de la izquierda). Entre las propiedades de los objetos de tipo Series tenemos index y values, que representan el ndice y los elementos de la serie:

In [4]:
s.index
Out[4]:
RangeIndex(start=0, stop=5, step=1)
In [5]:
s.values
Out[5]:
array([ 10,  20,  30,  40, -10], dtype=int64)

Una de las ventajas de trabajar con objetos de tipo Series, es que es posible indexar cada elemento de la serie con un valor descriptivo o etiqueta. Por ejemplo, si nuestros datos representan la tasa de natalidad de distintas ciudades espaolas, sera interesante usar el nombre de las ciudades cmo etiquetas del ndice, en lugar de usar etiquetas de tipo int:

In [6]:
nat = pd.Series([10.7, 7.5, 7.1, 17.8, 7.9 ],  
               index=['Murcia','Cantabria', 'Murcia', 'Melilla','Canarias' ],
               name = 'Tasa de Natalidad')
In [7]:
nat
Out[7]:
Murcia       10.7
Cantabria     7.5
Murcia        7.1
Melilla      17.8
Canarias      7.9
Name: Tasa de Natalidad, dtype: float64

El argumento name de la funcin pd.Series permite asignar un nombre a la serie creada. Otra de las propiedades de las Series es name:

In [8]:
nat.name
Out[8]:
'Tasa de Natalidad'

En este caso, tenemos dos procedimientos para acceder a los datos de una serie. El primero es mediante la etiqueta del ndice y notacin de corchetes, como en el caso de las listas o los arrays:

In [9]:
nat['Murcia']
Out[9]:
Murcia    10.7
Murcia     7.1
Name: Tasa de Natalidad, dtype: float64

El segundo es mediante la etiqueta del ndice (siempre que sea un nombre vlido en Python) como si fuera una propiedad de la serie:

In [10]:
nat.Murcia
Out[10]:
Murcia    10.7
Murcia     7.1
Name: Tasa de Natalidad, dtype: float64

Est claro que los datos los podramos haber representado con un array en lugar de una serie, pero como se puede ver, las series proporcionan mayor claridad en cuanto a los datos que contienen, la forma de acceso, etc.

En cuanto a las operaciones con series, se mantienen muchas de las propiedades vistas para los arrays de NumPy. Por ejemplo, las operaciones aritmticas (+, - , * , /) son vectorizadas, mantenindose el valor del ndice de la serie:

In [11]:
nat * 100
Out[11]:
Murcia       1070.0
Cantabria     750.0
Murcia        710.0
Melilla      1780.0
Canarias      790.0
Name: Tasa de Natalidad, dtype: float64

Las funciones definidas en la librera Numpy pueden ser utilizadas con series de Pandas. El argumento de la funcin puede ser una serie o un array.

La caracterstica estrella de las series es el alineamiento de datos en base al valor de los ndices en las operaciones entre dos series:

El resultado de la operacin anterior contiene el valor NaN. Este valor se usa para representar un campo vaco o desconocido. Es el equivalente al valor NULL del lenguaje SQL. En este caso, la serie nach tiene valor 8177 para la etiqueta Murcia, pero la serie nacm no contiene dicha etiqueta en su ndice, por lo que la suma de ambos produce como resultado NaN.

La clase DataFrame¶

La clase DataFrame est diseada para manejar datos representados en forma de tabla, donde tanto las filas como las columnas estn indexadas. Los objetos de tipo DataFrame pueden verse como una coleccin ordenada de columnas, cada una de las cuales viene representada por un objeto de tipo Series, con su nombre y su ndice. Cada columna tiene asociado un tipo de datos y todas las columnas comparten el msmo ndice. En este caso, usaremos el trmino dataframe para referirnos a los objetos de la clase DataFrame.

Hay varias formas de crear objetos de tipo DataFrame dependiendo de la procedencia de los datos.

  • Podemos usar la funcin pd.DataFrame si los datos se encuentran representados mediante una lista, un diccionario (de listas o de arrays), una Serie o incluso otro DataFrame.

  • Si los datos en encuentran en un fichero, Pandas proporciona las funciones pd.read_csv, pd.read_excel o pd.read_table para la creacin de DataFrame.

En el siguiente ejemplo usamos la funcin pd.DataFrame para crear un DataFrame a partir de un diccionario:

In [36]:
d = { 'Ciudades' : ['Melilla', 'Canarias', 'Galicia', 'Cantabria', 'Murcia'],
      'Nacidos H': [780, 8184, 10032, 2154, 8177] }


tabla = pd.DataFrame( data = d )
tabla
Out[36]:

Las claves del diccionario forman el ndice de las columnas. El valor de cada una de las claves del diccinario ha de ser una lista o un objeto de tipo ndarray de igual longuitud. Al igual que las Series, los atributos index y values representan el ndice y los datos del DataFrame respectivamente:

In [37]:

Out[37]:

In [38]:

Out[38]:
array([['Melilla', 780],
       ['Canarias', 8184],
       ['Galicia', 10032],
       ['Cantabria', 2154],
       ['Murcia', 8177]], dtype=object)

A diferencia de las series, que tienen un nico ndice, los dataframes tienen dos ndices. El primero es un ndice comn para todas las columnas y est asociado a cada una de las filas. El segundo ndice contiene un array de etiquetas o nombres de columnas. La propiedad columns representa el ndice de las columnas:

In [39]:
tabla.columns
Out[39]:
Index(['Ciudades', 'Nacidos H'], dtype='object')

La funcin pd.DataFrame admite ms argumentos, como por ejemplo los argumentos index y columns para crear el ndice de las filas y columnas del DataFrame respectivamente.
